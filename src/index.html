The Ultimate Arch Linux to BlackArch Guide

From Noob to Pro: A Comprehensive Journey

Maintained by Ch4120N Last updated: 2024

Introduction: Why Arch to BlackArch?

BlackArch Linux is an elite penetration testing distribution built on top of Arch Linux, offering over 2,800 security tools. Instead of downloading the BlackArch ISO (which can be buggy or overwhelming), the recommended way by the BlackArch team is to install a minimal, stable Arch system and then convert it.

Benefits of this approach:

  • Rock-solid base: You build a stable Arch system yourself.
  • Full control: You choose exactly which tools to install.
  • Lean system: No bloatware, only what you need.
  • Rolling release: Always get the latest tools via Arch's repositories.

Important Prerequisites

Warning: I strongly recommend you install Arch Linux inside a Virtual Machine first. This way, you won't accidentally erase your personal files or mess up your main operating system. When you've successfully installed Arch and converted it to BlackArch in the VM at least twice, then you can confidently do it on a real computer.

What is a Virtual Machine?

A Virtual Machine (VM) is like a "computer inside your computer". It uses software (like VirtualBox or QEMU) to create a fake PC where you can install and run operating systems without affecting your real machine. Think of it as a safe sandbox for learning and experimenting.

Step 0: Setting Up a Virtual Machine

If you're installing on real hardware, skip this step. Otherwise, follow these instructions:

1. Choose Your VM Software

  • VirtualBox (free, easy) - Download
  • QEMU/KVM (faster, more advanced) - usually preinstalled on Linux.

2. Create a New VM

  • Name: Arch Linux
  • Type: Linux / Arch Linux (or Other Linux 64-bit)
  • Memory (RAM): At least 4 GB (8 GB recommended)
  • Hard disk: 40 GB or more (dynamic allocation)
  • Network: NAT or Bridged
Tip: In VirtualBox, go to Settings > Storage and attach the Arch ISO file to the optical drive. Then boot the VM.

Step 1: Boot from the Arch Linux ISO

  1. Download the latest Arch ISO from archlinux.org.
  2. Burn it to a USB (or attach to VM) and boot from it.
  3. Select "Arch Linux install medium (x86_64, UEFI)" (or BIOS).
  4. Once booted, you'll be at a root shell prompt (root@archiso ~ #).

Pro tip: Change the keyboard layout if needed with loadkeys <layout> (e.g., loadkeys fr for French). The default is US.

Step 2: Connect to the Internet

You must have an internet connection to download packages.

Wired Ethernet

Usually works automatically. Test with:

ping -c 3 archlinux.org

WiFi

Use the interactive iwctl tool:

iwctl
# Inside the iwctl prompt:
device list                  # see your wireless device (e.g., wlan0)
station wlan0 scan
station wlan0 get-networks   # list available networks
station wlan0 connect "SSID" # replace SSID with your network name
exit

Step 3: Update the System Clock

Ensure your system clock is accurate to prevent package signature errors:

timedatectl set-ntp true
timedatectl status

Step 4: Partition the Disk

Identify your disk:

lsblk

Your main disk might be /dev/sda or /dev/nvme0n1. We'll refer to it as /dev/sdX.

Check UEFI: Run ls /sys/firmware/efi/efivars. If it exists, you're in UEFI mode.

Using fdisk

fdisk /dev/sda

Inside fdisk (for UEFI GPT):

  1. Type g for GPT.
  2. EFI: n -> 1 -> Enter -> +512M -> t -> 1
  3. Swap: n -> 2 -> Enter -> +4G -> t -> 2 -> 19
  4. Root: n -> 3 -> Enter -> Enter
  5. Write: w

Step 5: Format the Partitions

mkfs.fat -F32 /dev/sda1
mkswap /dev/sda2
swapon /dev/sda2
mkfs.ext4 /dev/sda3

Step 6: Mount the File Systems

mount /dev/sda3 /mnt
mkdir -p /mnt/boot
mount /dev/sda1 /mnt/boot

Step 7: Install the Base System

pacstrap /mnt base linux linux-firmware base-devel vim nano networkmanager

Step 8: Generate the Filesystem Table

genfstab -U /mnt >> /mnt/etc/fstab

Step 9: Chroot into the New System

arch-chroot /mnt

Step 10: Set the Time Zone

ln -sf /usr/share/zoneinfo/America/New_York /etc/localtime
hwclock --systohc

Step 11: Set the Locale

nano /etc/locale.gen # Uncomment en_US.UTF-8 UTF-8
locale-gen
echo "LANG=en_US.UTF-8" > /etc/locale.conf

Step 12: Set the Keyboard Layout

echo "KEYMAP=us" > /etc/vconsole.conf

Step 13: Set the Hostname

echo "blackarch-box" > /etc/hostname
nano /etc/hosts

Add to /etc/hosts:

127.0.0.1   localhost
::1         localhost
127.0.1.1   blackarch-box.localdomain   blackarch-box

Step 14: Set Root Password

passwd

Step 15: Network Configuration

systemctl enable NetworkManager

Step 16: Initramfs

mkinitcpio -P

Step 17: Install a Bootloader

For UEFI Systems

pacman -S grub efibootmgr
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg

For BIOS/Legacy Systems

pacman -S grub
grub-install --target=i386-pc /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg

Step 18: Exit, Unmount, and Reboot

exit
umount -R /mnt
reboot

Step 19: First Boot

Login as root with the password you set.

Step 20: Post-Installation (User & Sudo)

useradd -m -G wheel -s /bin/bash your_username
passwd your_username
pacman -S sudo
EDITOR=nano visudo

Uncomment %wheel ALL=(ALL:ALL) ALL in visudo.

Step 21: Install a Desktop Environment

Example for XFCE:

sudo pacman -S xfce4 xfce4-goodies lightdm
sudo systemctl enable lightdm

Step 22: Convert to BlackArch

Why Convert Instead of ISO?

Converting gives you a stable base, full control, and avoids the bloat and potential bugs of the full BlackArch ISO.

Step-by-Step Conversion

curl -O https://blackarch.org/strap.sh
echo "5ea40d49ecd00f61b0de78d19a6372d754a371c0 strap.sh" | sha1sum -c
chmod +x strap.sh
sudo ./strap.sh
sudo pacman -Syyu

Installing BlackArch Tools

Do not install all tools. Install by category:

pacman -Sg | grep blackarch
sudo pacman -S blackarch-webapp
sudo pacman -S blackarch-wireless

Conclusion

You've successfully installed Arch Linux and converted it into a BlackArch penetration testing environment. Remember to use these tools ethically and legally.
